# 12 · Glossary

← [11 · Operations Runbook](11-operations-runbook.md) · [Wiki home](README.md)

---

Every term used in this wiki, defined plainly. Chapter references point to fuller explanations.

- **ASP.NET Core** — the .NET framework for building web servers/APIs (Kestrel, middleware, DI, routing). [07](07-technologies-explained.md)
- **Async / await** — language feature that lets a method pause on I/O and free its thread to do other work; resumes when the I/O completes. [08](08-concurrency-and-internals.md)
- **AsyncLocal** — state that follows the logical async flow (not one thread); how `Activity.Current` survives `await`. [08](08-concurrency-and-internals.md)
- **Azure** — Microsoft's cloud provider. [02](02-foundational-concepts.md)
- **Azure Container Apps** — managed service that runs container images as long-lived services with HTTPS, TLS, custom domains, revisions. Hosts the API + Redis. [10](10-deployment.md)
- **SQLite (embedded)** — lightweight relational database that runs in-process as a single file; CacheScope's L4 source of truth, created + seeded on boot with no separate DB service or cost. [04](04-the-five-cache-layers.md)
- **Backpressure** — a mechanism to prevent a fast producer from overwhelming a slow consumer; here, a bounded drop-oldest channel. [08](08-concurrency-and-internals.md)
- **Bicep** — Azure's Infrastructure-as-Code language; declares the cloud resources to create. [07](07-technologies-explained.md)
- **BackgroundService / hosted service** — a long-running task .NET starts at boot (e.g. the broadcaster, the write-behind flusher). [08](08-concurrency-and-internals.md)
- **Cache** — a fast, nearby copy of data that normally lives somewhere slow. [01](01-introduction-and-business-problem.md)
- **Cache-aside** — the pattern where the app checks the cache, and on a miss loads from the DB and populates the cache itself. [04](04-the-five-cache-layers.md)
- **Cache-Control** — HTTP header telling clients/CDNs how long they may cache a response. [02](02-foundational-concepts.md)
- **Cache invalidation** — removing/replacing a cached copy because the underlying data changed. [01](01-introduction-and-business-problem.md)
- **Cache stampede / thundering herd** — many concurrent requests missing the same key at once and flooding the DB. Fixed by single-flight. [04](04-the-five-cache-layers.md)
- **CDN** — Content Delivery Network; global edge servers that cache content near users (Cloudflare = L0). [02](02-foundational-concepts.md)
- **CF-Cache-Status** — Cloudflare header reporting whether the edge served the request (HIT/MISS/…). [04](04-the-five-cache-layers.md)
- **Channel (`System.Threading.Channels`)** — a thread-safe in-memory producer/consumer queue. [08](08-concurrency-and-internals.md)
- **CI/CD** — Continuous Integration / Continuous Deployment; automated test+build+deploy (GitHub Actions). [07](07-technologies-explained.md)
- **Client / Server** — the requester / the responder. [02](02-foundational-concepts.md)
- **Cloud** — renting computers/services over the internet instead of owning hardware. [02](02-foundational-concepts.md)
- **Cloudflare** — CDN + DNS + edge + TLS + static hosting; provides L0 and hosts the frontend. [07](07-technologies-explained.md)
- **Composition root** — the single place (here `Program.cs`) that wires all dependencies together. [03](03-system-architecture.md)
- **ConcurrentDictionary** — a thread-safe dictionary usable by many threads at once. [08](08-concurrency-and-internals.md)
- **Container** — a running instance of a container image; an isolated, portable unit. [02](02-foundational-concepts.md)
- **Container image** — the read-only blueprint (app + dependencies) that runs as a container. [02](02-foundational-concepts.md)
- **CORS** — Cross-Origin Resource Sharing; rules letting a browser call an API on a different origin. [05](05-request-lifecycle.md)
- **Correlation id** — an id tying one request's logs, trace, and dashboard row together. [05](05-request-lifecycle.md)
- **Database** — a program that stores data persistently and answers queries. [02](02-foundational-concepts.md)
- **DbContext** — EF Core's typed gateway to the database. [06](06-code-walkthrough.md)
- **Dependency Injection (DI)** — supplying a class's collaborators from a central container instead of the class creating them. [03](03-system-architecture.md)
- **Dependency Inversion** — modules depend on abstractions (interfaces in `Shared`), not on each other. [03](03-system-architecture.md)
- **DNS** — the internet's phone book; maps domain names to IP addresses. [02](02-foundational-concepts.md)
- **Docker** — tool to build/run containers. [02](02-foundational-concepts.md)
- **Dockerfile** — the recipe to build a container image (multi-stage here). [07](07-technologies-explained.md)
- **Edge** — servers physically close to users (Cloudflare's PoPs); where L0 caches. [04](04-the-five-cache-layers.md)
- **Entity Framework Core (EF Core)** — .NET ORM; query the DB with objects/LINQ; here over SQLite, schema built on boot with `EnsureCreatedAsync()` + seeding. [07](07-technologies-explained.md)
- **ETag** — a version tag on a response enabling cheap revalidation (`304 Not Modified`). [02](02-foundational-concepts.md)
- **Eviction** — removing an entry from a cache (TTL expiry or memory pressure). [02](02-foundational-concepts.md)
- **Garbage collector** — the .NET component that automatically frees unused memory. [02](02-foundational-concepts.md)
- **GHCR** — GitHub Container Registry; where the image is stored and Azure pulls it from. [10](10-deployment.md)
- **Hit / Miss** — data found in the cache / not found. [02](02-foundational-concepts.md)
- **HTTP** — the request/response protocol of the web (methods, headers, status codes, bodies). [02](02-foundational-concepts.md)
- **IMemoryCache** — .NET in-process memory cache; L2. [04](04-the-five-cache-layers.md)
- **Index (database)** — a structure that makes row lookups fast; the primary key `Id` is indexed. [02](02-foundational-concepts.md)
- **Interlocked** — atomic, lock-free operations on a number (used for counters). [08](08-concurrency-and-internals.md)
- **IP address / Port** — a machine's network address / a numbered service door on it. [02](02-foundational-concepts.md)
- **IServiceScopeFactory** — lets a singleton create a fresh DI scope to safely use scoped services in the background. [08](08-concurrency-and-internals.md)
- **JSON** — a text format for structured data; the web API wire format. [02](02-foundational-concepts.md)
- **Kestrel** — the HTTP server built into ASP.NET Core. [07](07-technologies-explained.md)
- **L0–L4** — the five cache layers: Cloudflare edge, browser, memory, Redis, database. [04](04-the-five-cache-layers.md)
- **Latency / Throughput** — time per operation / operations per second. [02](02-foundational-concepts.md)
- **Lazy** — a value computed once on first access, even under concurrent access; key to single-flight. [08](08-concurrency-and-internals.md)
- **Middleware** — an ordered component in the request pipeline that can act before/after the next. [07](07-technologies-explained.md)
- **`EnsureCreatedAsync()` (EF Core)** — builds the DB schema directly from the model at startup (no migrations); CacheScope uses it to create the embedded SQLite DB on boot. [07](07-technologies-explained.md)
- **Modular monolith** — one deployable process, split into strict compile-enforced modules. [03](03-system-architecture.md)
- **.NET / C#** — the backend runtime/platform / the language. [02](02-foundational-concepts.md)
- **OpenTelemetry (OTel)** — vendor-neutral standard for traces/metrics/logs; built on `Activity`. [07](07-technologies-explained.md)
- **ORM** — Object-Relational Mapper; query a DB via language objects (EF Core). [02](02-foundational-concepts.md)
- **PaaS** — Platform-as-a-Service; a managed service the provider operates (e.g. Azure Container Apps). [02](02-foundational-concepts.md)
- **Percentile (P50/P95/P99)** — the value below which that % of latencies fall; computed via a histogram. [08](08-concurrency-and-internals.md)
- **Primary key** — the unique, indexed identifier column of a table (`Id`). [02](02-foundational-concepts.md)
- **Process / Thread** — a running program with private memory / a single line of execution within it. [02](02-foundational-concepts.md)
- **RAM / Disk** — fast volatile memory / slower persistent storage. [02](02-foundational-concepts.md)
- **Redis** — in-memory key-value store used as the shared distributed cache (L3). [07](07-technologies-explained.md)
- **Registry** — a store for container images (GHCR). [02](02-foundational-concepts.md)
- **RequestTrace / RequestDetail** — the lean streamed record / the full record (with waterfall + trace id) of a request. [06](06-code-walkthrough.md)
- **Resource Timing API** — browser API used to measure L1 (browser-cache) hits (`transferSize === 0`). [09](09-ui-features.md)
- **Revision (Container Apps)** — a versioned instance of the running app; a deploy creates a new one. [10](10-deployment.md)
- **Serialization / Deserialization** — converting an object to/from a portable format (JSON) to cross a process/network boundary. [02](02-foundational-concepts.md)
- **Signal (Angular)** — reactive state that re-renders the UI when it changes. [09](09-ui-features.md)
- **SignalR** — .NET realtime library for server→client push over WebSockets. [07](07-technologies-explained.md)
- **Single-flight** — coalescing concurrent misses for one key into a single DB load. [08](08-concurrency-and-internals.md)
- **Singleton / Scoped / Transient** — DI lifetimes: one per process / one per request / one per resolution. [08](08-concurrency-and-internals.md)
- **SPA** — Single-Page Application; loads once, updates dynamically (the dashboard). [02](02-foundational-concepts.md)
- **SQL** — the query language for relational databases. [02](02-foundational-concepts.md)
- **Span / ActivitySource / Activity** — one timed unit of a trace / the tracer / the .NET span object. [07](07-technologies-explained.md)
- **Stampede protection** — see single-flight. [08](08-concurrency-and-internals.md)
- **Status code** — HTTP result number (200/304/404/500). [02](02-foundational-concepts.md)
- **TCP / TLS / HTTPS** — reliable connection / encryption / secure HTTP. [02](02-foundational-concepts.md)
- **Timing-Allow-Origin** — header letting a cross-origin page read Resource Timing details (needed for L1). [11](11-operations-runbook.md)
- **TTL** — Time To Live; how long a cached copy may live. [02](02-foundational-concepts.md)
- **WebSocket** — a persistent two-way connection enabling server push (SignalR uses it). [02](02-foundational-concepts.md)
- **Write strategies** — cache-aside / write-through / write-behind / refresh-ahead; how writes keep caches honest. [04](04-the-five-cache-layers.md)
- **X-Served-By** — response header naming the layer that served the request. [05](05-request-lifecycle.md)
- **Zipf distribution** — a power-law traffic pattern where a few "hot" keys get most requests. [09](09-ui-features.md)
